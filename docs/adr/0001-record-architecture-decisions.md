# ADR 0001 — 아키텍처 결정을 ADR로 기록한다

- 상태: Accepted
- 일자: Sprint 0 (설립)

## 배경
헌법은 **Documentation First**를 요구한다. 설계 결정의 *이유*가 코드에 묻히면 안 된다.

## 결정
모든 중대한 아키텍처/기술 결정은 `docs/adr/NNNN-title.md`에 ADR로 기록한다.
형식: 배경 → 결정 → 근거 → 대안 → 결과.

## 결과
- 결정의 추적성 확보. 신규 기여자(사람/AI)가 *왜*를 알 수 있다.
- CTO 자율 결정도 ADR로 남겨 CEO가 사후 검토 가능.
