# Atlas Alpha Build Fix #001 보고 (CTO)

> 문제: Alpha가 "채팅앱"처럼 보여 사용자가 계속 질문하게 되고, 자료를 제공할 방법이 없었다.
> 수정: **채팅 → CEO Dashboard + 업무 카드** 중심으로 전환. 자료 제공 입력·상태 전이·결과 카드·추천 채용 추가. (AI 호출 0 유지)

## 1. 변경한 UX 구조
- 메인을 **CEO Dashboard**로 전환. 채팅 로그 제거.
- "무엇을 도와드릴까요?" → **새 업무 등록(한 줄 입력)** + 우선순위 빠른 시작.
- 모든 것이 **카드 + 명확한 CTA**(토스식). 내부 지표(Skill/Matching/Confidence 점수/Token/Cost) 미노출.

## 2. 대시보드 섹션
1. 인사 + 회사명 + **운영 준비도 %** (막대) — 정보·직원 보유율로 계산(AI 없음)
2. **오늘의 우선순위** — open 업무, 없으면 빠른 시작 제안(신메뉴 홍보/리뷰 답변/재고 점검)
3. **새 업무 시작** — 한 줄 입력 → 업무 카드 생성
4. **진행 중 업무** — 카드(상태·담당·필요 자료·추천 채용·결과·승인/수정)
5. **현재 직원** — 대표 비서 출근 + 직원 출근/미채용
6. **추천 채용** — open 업무 부족 직군 + 이유 + 채용 CTA
7. **직원 채용 목록** — 5종(Writer/Designer/Marketing/Operations/CS): 역할·할 수 있는 일·상태·CTA

## 3. 자료 제공 방식 (치명 결함 수정)
업무 카드의 "필요 자료"마다 입력 제공:
- **텍스트 / URL / 파일 / 이미지**(파일·이미지는 파일명·타입 메타데이터로 mock 저장)
- 제공 시 → `Material` 기록(업무·요청 맥락) + **Company/Brand Memory에 축적** → 같은 자료 필요한 다른 업무도 자동 갱신
- 제공 즉시 **업무 상태 갱신**.

## 4. 업무 카드 상태 흐름
```
needs_hire(직원 필요) ─┐
awaiting_materials(자료 대기) → ready(실행 가능) → [실행] → delivered(결과 완료) → approved / revise
```
- 추측 금지: 자료 부족(Confidence<70)이면 결과물 미생성 → 자료 요청.
- 부족 직원은 **비차단 추천**(보유 직원으로 가능한 부분은 실행).
- 수정 요청 → 결과 초기화 → 재실행 가능.

## 5. 결과물 카드 방식 (AI 없음 = mock)
- 결과물 카드: 유형·담당 직원·상태(최종본/초안)·**mock/placeholder 내용**·승인/수정.
- 텍스트형(text·social_post·ad_copy·report·customer_reply)만 mock 생성.
- 이미지/영상은 **실제 생성하지 않음** → "추후 지원"(Designer 필요 시 채용 추천).

## 6. 직원 추천 / 채용 목록
- 고정 카탈로그 5종 + 보유 여부. 부족 직군은 추천 채용에 노출(예: 콘텐츠 요청 → Designer 추천).
- 채용 즉시 직원 출근 + 관련 업무 상태 재계산.

## 7. AI 호출 비용 절감 방식
- 이번 Alpha **AI 호출 0**. 의도 분석=규칙, 결과물=템플릿 mock.
- 대시보드·직원 목록·자료 업로드·상태·CTA·진행률·결과 목록은 **일반 앱 로직**.
- AI 개입 지점은 향후(무료 진단/업무 분석/결과물 생성)로 한정 — 카드/폼/버튼 중심으로 대화 비용 최소화.
